# معمارية مشروع نظام إدارة مدرسة أبي دجانة المتوسطة (Project Architecture)

هذا الملف يحتوي على الخريطة الكاملة لهيكل المشروع ودور كل مجلد وملف فيه، لتسهيل التعديل والتطوير السريع وفقاً لمبادئ **Clean Architecture** والملفات ذات المسؤولية الفردية (Single Responsibility).

---

## هيكل المجلدات الرئيسي (Directory Structure)

```text
├── public/                     # الشعارات والصور الثابتة للموقع

│   ├── app/                    # صفحات ومسارات تطبيق Next.js (App Router)
│   │   ├── layout.js           # التنسيق الرئيسي للتطبيق وتحميل الخطوط
│   │   ├── page.js             # صفحة تسجيل الدخول الرئيسية المشتركة

│   │   │   ├── page.js         # لوحة تحكم المدير الرئيسية (الداشبورد)
│   │   │   ├── reports/        # صفحة استعراض وطباعة وتصدير التقارير
│   │   │   ├── settings/       # إعدادات المدرسة، أوقات الدوام، والحقول المخصصة
│   │   │   └── teachers/       # شاشة تفاصيل وإدارة المعلمين للمدير
│   │   └── employee/           # صفحات موظف التسجيل وصلاحياته
│   │       ├── page.js         # شاشة التحضير اليومي
│   │       └── teachers/       # شاشة إدارة وإضافة المعلمين

│   ├── components/             # المكونات الرسومية القابلة لإعادة الاستخدام (UI Components)
```

## أدوار الملفات الأساسية بالتفصيل

### 1. طبقة منطق العمل (core)
- `lateness.js`: يقوم باستقبال وقت حضور المعلم (مثال: "08:15") ووقت بداية الدوام (مثال: "07:00") ويقوم بعملية الطرح وإرجاع دقائق التأخير بدقة كقيمة رقمية.
- `calendar.js`: يحتوي على دوال للتحويل بين التاريخ الميلادي والهجري وعرض الأشهر الهجرية الرسمية، متطابقاً مع تقويم أم القرى.
- `validation.js`: يفحص قائمة المعلمين لتحديد ما إذا كان هناك غياب مريب لليوم الحالي (غياب > 30% أو غياب متتالي لمعلم لأكثر من 3 أيام) لتنبيه المدير.

### 2. طبقة الخدمات (services)
- `db.js`: يحتوي على الدوال البرمجية (مثل `getTeachers`, `saveAttendance`, `requestLeave`, `updateSettings`) للتعامل مع قاعدة بيانات Supabase.
- `excel.js`: يستقبل بيانات التقارير المجمعة أو الفردية ويقوم بتوليد شيت Excel منسق بصفوف ملونة وخلايا مرتبة تعبر عن أيام الحضور والتأخير ومجموع الساعات.
- `holidays.js`: يقوم بالربط مع الـ API لجلب جدول الإجازات الرسمية للمملكة العربية السعودية وتحديثها في قاعدة البيانات.

### 3. طبقة الواجهات والصفحات (app)
- يتم تنظيمها بملفات صغيرة وموجزة لاستعراض البيانات وتمرير الإجراءات إلى طبقة الـ `services` والـ `core`.
- تعتمد كلياً على المكونات الموجودة في `components` لتحقيق كود نظيف وقابل للقراءة والصيانة السريعة.
